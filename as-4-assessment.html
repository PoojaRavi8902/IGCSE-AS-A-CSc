Part B — Short Questions (15)

Write brief answers in your notebook. Model answers will be revealed when you press Reveal Short Answers.

  1. 1. Define a Boolean proposition and give one example.
    A Boolean proposition is a statement that can only be TRUE or FALSE. Example: "The temperature is above 25°C." (Either true or false.)
  2. 2. State the truth table for A AND B.
    A AND B truth table (A,B → A AND B): 00→0, 01→0, 10→0, 11→1.
  3. 3. Write the Boolean expression for: "Switch on the heater if it is cold OR humidity is high."
    Heater = Cold OR High_Humidity.
  4. 4. What does NOT(A OR B) equal by De Morgan's law?
    NOT(A OR B) = (NOT A) AND (NOT B).
  5. 5. Give one practical example of an XOR gate.
    XOR can be used for "odd parity" checking — output 1 if an odd number of inputs are 1 (e.g., simple error-detection/parity circuits).
  6. 6. How do NAND and NOR gates relate to AND/OR gates?
    NAND is the complement of AND (NAND = NOT(AND)). NOR is the complement of OR (NOR = NOT(OR)).
  7. 7. What is the minimal number of rows for a 2-input truth table? How many for 4 inputs?
    For 2 inputs: 4 rows (2^2). For 4 inputs: 16 rows (2^4).
  8. 8. Explain briefly why NAND alone is useful in circuit design.
    NAND is functionally complete: any Boolean function can be implemented using only NAND gates, which simplifies manufacturing and design.
  9. 9. Convert the expression X = A AND (B OR C) into a short verbal problem statement.
    "X is true if A is true and either B or C (or both) are true." Example: "Start engine if ignition is on AND (brake pressed OR gear in neutral)."
  10. 10. What is an intermediate workspace column in a truth table and why is it useful?
    An intermediate workspace column records the output of sub-parts of a circuit (e.g., M = A AND B) so you can compute the final output step-by-step and avoid mistakes.
  11. 11. State the truth table row count formula for n inputs.
    Number of rows = 2^n (two to the power of number of inputs n).
  12. 12. Give the truth table for XOR (A XOR B).
    A XOR B: 00→0, 01→1, 10→1, 11→0.
  13. 13. Why might a designer prefer to draw a circuit symbol rather than write an expression?
    Circuits provide a visual representation that makes flows, intermediate signals and hardware mapping clearer and easier to implement/verify.
  14. 14. What is the output of NOR when inputs are A=0 and B=0?
    NOR outputs 1 when both inputs are 0, so output = 1.
  15. 15. Give the Boolean expression that corresponds to: "Alarm if window open AND (intruder sensor OR break-glass sensor)."
    Alarm = Window_open AND (Intruder_sensor OR Break_glass_sensor).
Part C — Exam-style Questions (15)

Write longer answers in the textareas. Model solutions are hidden and will be revealed when you press Reveal Model Answers. Your typed answers will remain for teacher review.

  1. 1. Convert the following problem statement into a Boolean expression and then draw the equivalent logic circuit (describe it in words if drawing is not possible here): "Turn on the garden lights if it is dark AND (motion is detected OR manual override is ON)."
    Model solution: Expression: Lights = Dark AND (Motion OR Override). Circuit (description): An OR gate takes Motion and Override as inputs; its output and Dark feed into an AND gate; AND output is Lights. Truth table: Lights = 1 only when Dark=1 and at least one of Motion or Override =1.
  2. 2. For A, B, C inputs, construct the truth table for X = (A AND B) OR (NOT C). Show all 8 rows and final X column.
    Model solution (final X column): Rows (A B C) → X: 000 → 1 (NOT C =1) 001 → 0 (NOT C =0, A AND B =0) 010 → 1 (NOT C =1) 011 → 0 100 → 1 101 → 0 110 → 1 (A AND B =1) 111 → 0 (Explanation: compute NOT C, compute A AND B, then OR.)
  3. 3. Show step-by-step how to convert the truth table below to a sum-of-products expression (use the rows that produce 1). Table: inputs A B C; outputs X: rows with 1 are (0,0,1), (0,1,1), (1,0,0).
    Model solution: Rows ⇒ minterms: (0,0,1) → NOT A AND NOT B AND C (0,1,1) → NOT A AND B AND C (1,0,0) → A AND NOT B AND NOT C Combine with OR: X = (NOT A AND NOT B AND C) OR (NOT A AND B AND C) OR (A AND NOT B AND NOT C). Optionally factor NOT A AND C from first two terms: X = (NOT A AND C AND (NOT B OR B)) OR (A AND NOT B AND NOT C) = (NOT A AND C) OR (A AND NOT B AND NOT C).
  4. 4. Use De Morgan's laws to simplify NOT(A OR (B AND C)). Show each step.
    Model solution: Start: NOT(A OR (B AND C)) De Morgan: = (NOT A) AND (NOT (B AND C)) Apply De Morgan to second: NOT(B AND C) = (NOT B) OR (NOT C) So final: = (NOT A) AND ((NOT B) OR (NOT C)) Optionally distribute: = (NOT A AND NOT B) OR (NOT A AND NOT C).
  5. 5. Design a truth table and logic expression for: "Safety alarm sounds if both the door sensor and the window sensor trigger, unless the system is in maintenance mode (maintenance disables alarm)." (Inputs: Door, Window, Maintenance)
    Model solution: Expression: Alarm = (Door AND Window) AND (NOT Maintenance) Truth table rows where Alarm =1: Door=1,Window=1,Maintenance=0 → Alarm=1. All other combinations Alarm=0. You can also write Alarm = NOT(Maintenance) AND Door AND Window.
  6. 6. Explain why NAND and NOR are called universal gates and give a simple example of implementing NOT using NAND only.
    Model solution: Universal gates: NAND and NOR are functionally complete — any Boolean function can be built using only NANDs or only NORs. Example: NOT A using NAND: connect both inputs of a NAND gate to A → output = NOT(A AND A) = NOT A. So single NAND with tied inputs produces inversion.
  7. 7. A circuit has intermediate nodes M = A AND B, N = NOT C, P = M OR N. Explain how to fill a truth table with workspace columns and why partial checking (e.g., all rows with C=0) can speed verification.
    Model solution: Include columns for A,B,C,M,N,P. Compute M from A,B; N from C; P from M,N. Partial checks: if some path forces output 0 for a whole subset (e.g., if C=0 sets N=1 making P=1 regardless of M), you can inspect fewer rows. Using workspace reduces mistakes and clarifies the circuit flow.
  8. 8. Given expression X = (A XOR B) XOR C, show whether X is true for inputs A=1,B=1,C=1 and explain parity interpretation.
    Model solution: Compute A XOR B = 1 XOR 1 = 0. Then 0 XOR C = 0 XOR 1 = 1. So X = 1. Parity: XOR chain gives 1 when an odd number of inputs are 1 (odd parity). Here 3 ones → odd → output 1.
  9. 9. Simplify the expression: (A AND B) OR (A AND NOT B) — show algebraic steps.
    Model solution: Factor A: = A AND (B OR NOT B) Since (B OR NOT B) = 1, expression = A AND 1 = A. So (A AND B) OR (A AND NOT B) simplifies to A.
  10. 10. Create a logic expression for: "System OK if sensor1 AND sensor2 are OK, OR if manual override is active (override bypasses sensors)." Identify when system is OK.
    Model solution: Expression: OK = (Sensor1 AND Sensor2) OR Override. System OK when either both sensors report OK, or whenever Override=1 regardless of sensors. Truth table shows OK=1 for rows with Override=1, and when both sensors=1 and Override=0.
  11. 11. Convert the truth-table below to a minimized logic expression (assume variables A,B,C): Rows producing 1: (0,0,1), (0,1,1), (1,0,0) Provide the minimized expression if possible.
    Model solution: From earlier (Part C Q3), X = (NOT A AND C) OR (A AND NOT B AND NOT C). This is already simplified by factoring common terms. Further algebraic simplification depends on K-map; final minimized expression is X = (NOT A AND C) OR (A AND NOT B AND NOT C).
  12. 12. Explain, with a short example, how you would test a student's circuit solution for correctness under exam conditions (systematic checklist).
    Model solution: Checklist: 1) Verify expression matches problem statement (conditions captured). 2) Build truth table: compute for all input combinations. 3) Check circuit: compute intermediate nodes for each row and final output. 4) Compare truth table from expression, circuit and expected outputs. Example: For Lights = Dark AND (Motion OR Override), test rows where Dark=0 => Lights must be 0 regardless; test rows where Dark=1 and Motion or Override=1 => Lights=1.
  13. 13. Show how to implement XNOR (equivalence) using basic gates (AND, OR, NOT) and give the expression.
    Model solution: XNOR outputs 1 when inputs equal. Expression: XNOR = (A AND B) OR (NOT A AND NOT B). Alternatively XNOR = NOT(A XOR B). Using AND, OR, NOT: (A AND B) OR ((NOT A) AND (NOT B)).
  14. 14. A logic circuit is given: X = NOT(A AND (B OR C)). Use De Morgan to rewrite it and identify a circuit using only NAND gates if possible (describe how).
    Model solution: Rewrite: NOT(A AND (B OR C)) = (NOT A) OR (NOT (B OR C)) = (NOT A) OR ((NOT B) AND (NOT C)). Using NANDs: A AND (B OR C) can be built and then NAND used to invert. Alternatively use NAND form by applying De Morgan to transform OR into NAND combinations. Specific construction: construct B OR C via NAND: (NOT(B NAND B) NAND NOT(C NAND C)) etc. Practical approach: implement B OR C with NANDs and then NAND with A. (Exam marking expects De Morgan steps and reasoning how NANDs can implement inversion.)
  15. 15. Provide a short worked example (with workspace) converting the truth table: A B C → X 0 0 1 → 1 0 1 1 → 1 1 0 0 → 1 others → 0 into a minimized circuit and explain each step.
    Model solution: 1) List minterms corresponding to 1s: m1: NOT A AND NOT B AND C m2: NOT A AND B AND C m3: A AND NOT B AND NOT C 2) Factor first two: NOT A AND C AND (NOT B OR B) → NOT A AND C 3) So expression: X = (NOT A AND C) OR (A AND NOT B AND NOT C) 4) Implement circuit: one AND for NOT A & C, another 3-input AND for A & NOT B & NOT C, then OR the results. Workspace columns: NOT A, NOT B, NOT C, (NOT A AND C), (A AND NOT B AND NOT C), final OR → X. This is the minimized circuit (no further common factoring possible).
