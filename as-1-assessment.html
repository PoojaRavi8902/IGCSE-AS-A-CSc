Section B — Short Questions (15)

Write short answers. When you click Submit, model answers will appear below each question for self-checking.

  1. 1. Explain why computers use binary rather than denary internally.

    Answer: Computers use binary because hardware components (switches, transistors) reliably represent two states (on/off). Binary simplifies circuit design and error detection and maps directly to physical states.

  2. 2. Convert binary 101101 to denary.

    Answer: 1×32 +0×16 +1×8 +1×4 +0×2 +1×1 = 32+8+4+1 = 45.

  3. 3. Give the hexadecimal representation of binary 11110000.

    Answer: 1111 0000₂ = F0₁₆.

  4. 4. What is a byte and how many possible values can it hold?

    Answer: A byte is 8 bits and can represent 2⁸ = 256 distinct values (0–255 unsigned).

  5. 5. State Nyquist theorem in one sentence.

    Answer: To reconstruct a signal without aliasing, sample at at least twice the highest frequency present in the signal.

  6. 6. What is colour depth (bit depth)?

    Answer: Colour depth is the number of bits used per pixel (or per primary channel), determining how many colours each pixel can represent.

  7. 7. Give one advantage of vector graphics.

    Answer: Vector images scale without loss of quality because objects are stored as geometric descriptions rather than pixels.

  8. 8. What is run-length encoding (RLE)?

    Answer: RLE replaces consecutive repeated values with a pair indicating the value and the count, effective where long runs occur.

  9. 9. How are Unicode code points shown in documentation?

    Answer: As U+ followed by a hexadecimal number, e.g., U+0041 for 'A'.

  10. 10. Why might BCD be used for currency values?

    Answer: BCD stores each decimal digit separately, avoiding rounding errors that arise when using binary floating-point for fixed decimal currency amounts.

  11. 11. Explain the difference between colour depth and bit depth (as used per channel).

    Answer: Colour depth often means total bits per pixel (e.g., 24-bit). Bit depth per channel is bits used for each colour channel (e.g., 8 bits per R, G, B).

  12. 12. What is a memory dump and why is hex used?

    Answer: A memory dump shows raw bytes in memory; hex is compact and maps neatly to bytes (2 hex digits per byte), making it easier to read than binary.

  13. 13. How many distinct codes does 7-bit ASCII provide?

    Answer: 2⁷ = 128 codes (0–127).

  14. 14. Briefly explain two's complement conversion trick (flip bits after first 1).

    Answer: To find two's complement: from LSB move left past any zeros up to the first 1 (leave it), then flip all remaining higher bits; this yields the negative representation.

  15. 15. What effect does increasing sampling resolution (bits per sample) have on audio?

    Answer: Higher resolution reduces quantization error and increases dynamic range (better fidelity), but increases file size.

Section C — Exam-Style Questions (15)

Attempt these as full answers. Use the textareas to type your responses. When you click Submit, model answers will be revealed beneath each question for guidance and grading.

  1. 1. Explain place value in the denary system with an example (use 346).

    Model answer: Place value assigns a power of 10 to each digit position. For 346: 3×10² = 300, 4×10¹ = 40, 6×10⁰ = 6. The total value is 300 + 40 + 6 = 346. The rightmost digit is least significant; place values increase to the left.

  2. 2. Convert the denary number 78 to binary and show steps.

    Model answer: Largest power of 2 ≤ 78 is 64 (2⁶) → 1 at 2⁶ place. Remainder 14. Next power 8 (2³) → 1 at 2³. Remainder 6 → 1 at 2² (4), remainder 2 → 1 at 2¹, remainder 0. Binary: 1001110₂.

  3. 3. Describe how two's complement represents negative numbers and convert -79 to 8-bit two's complement.

    Model answer: Two's complement: invert all bits of magnitude and add 1. For +79: 01001111. Invert: 10110000. Add 1 → 10110001. So -79 is 10110001 (8-bit).

  4. 4. Explain why UTF-8 is backward compatible with ASCII and outline the byte patterns used.

    Model answer: UTF-8 encodes ASCII characters (U+0000 to U+007F) as single bytes with MSB 0, identical to 7-bit ASCII. Multi-byte sequences start with leading bits 110,1110,11110 and continuation bytes have 10xxxxxx, ensuring no ambiguity and backward compatibility.

  5. 5. Calculate the approximate file size (in bytes and MiB) of a 1366×768 image at 24-bit colour (show calculation).

    Model answer: Bits = 1366×768×24 = 25,178,112 bits. Bytes = ÷8 = 3,147,264 bytes. In MiB ≈ 3,147,264/1024/1024 ≈ 3.00 MiB (≈3 MiB).

  6. 6. Describe run-length encoding and give a short example of compressing 'AAAAABBBCC'.

    Model answer: RLE replaces runs with (count, value). 'AAAAABBBCC' → (5,A)(3,B)(2,C). Depending on scheme bytes could be: 05 A 03 B 02 C; reduces repeated sequences.

  7. 7. Explain the difference between lossy and lossless compression and give one example of each and why each is suitable.

    Model answer: Lossless: original recovered exactly (e.g., PNG, ZIP) — used for text/data. Lossy: some information discarded to reduce size (e.g., JPEG, MP3) — used where human perception tolerates small differences like photos/audio.

  8. 8. Explain how quantization error arises in ADC and how increasing resolution affects it.

    Model answer: Quantization error is the difference between the true analogue amplitude and the nearest discrete level chosen. Increasing bit resolution gives more levels, reducing quantization error and improving fidelity, at cost of larger data size.

  9. 9. Show binary addition of 0110 and 1011 and state if overflow occurs for 4-bit storage (unsigned).

    Model answer: 0110 (6) + 1011 (11) = 10001 (17). Stored in 4 bits would be 0001 with a carry/overflow. For 4-bit unsigned overflow occurs because 17 > 15.

  10. 10. Describe how BCD addition is corrected when a nibble exceeds 9 (briefly demonstrate).

    Model answer: When a nibble > 9, add correction 0110 (decimal 6) to that nibble; carry out any carry to next nibble. This converts invalid BCD results to valid decimal digits with correct carries.

  11. 11. What are the benefits and limitations of using UTF-8 vs fixed-width encodings for international text?

    Model answer: UTF-8 benefits: backward compatible with ASCII, space-efficient for ASCII text, supports whole Unicode. Limitations: variable byte length complicates indexing. Fixed-width (UTF-32) simplifies indexing but wastes space for common texts and increases memory use.

  12. 12. Explain why computer scientists introduced binary prefixes (Ki, Mi, Gi) and show conversion of 4 GiB to MiB.

    Model answer: Binary prefixes clarify 2ⁿ-based sizes: 1 Ki = 2¹⁰. 4 GiB (assuming Gi = 2³⁰) = 4 × 1024 MiB = 4096 MiB. They avoid ambiguity between 10³ and 2¹⁰ multipliers.

  13. 13. Give a short explanation of why a bitmap image may show banding if bit depth is too low.

    Model answer: Low bit depth reduces number of colours/levels, causing large steps between adjacent colours so gradual gradients become visible as bands rather than smooth transitions.

  14. 14. Outline the process of converting a hex value 2A6 to decimal (brief calculation).

    Model answer: 2×16² + A(10)×16 + 6 = 2×256 + 10×16 + 6 = 512 + 160 + 6 = 678.

  15. 15. Describe how Huffman coding ensures no code is a prefix of another (prefix property) and why that matters.

    Model answer: Huffman builds a prefix-free binary tree so each symbol is a leaf: no code is the prefix of another. This ensures unique decodability: the bit stream can be parsed unambiguously from left to right.
