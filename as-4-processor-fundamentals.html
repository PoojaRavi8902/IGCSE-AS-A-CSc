This chapter focuses on Boolean logic, operators, truth tables, and logic circuits. These concepts are the foundation for how processors make decisions and how digital systems are designed.

4.01 Boolean Logic and Problem Statements

A Boolean proposition (logic assertion) is a statement that can only be TRUE or FALSE.

Examples:

  • “Colombo is further north than Singapore.” → TRUE
  • “It is raining.” → TRUE or FALSE
  • “The temperature is above 25°C.” → TRUE or FALSE

Everyday conditions can be converted into Boolean statements, which can then be combined into problem statements. For example:

  • “Take an umbrella IF it is raining OR rain is forecast.”
  • “Air conditioning turns on IF office hours AND temperature > 25°C.”
Exam Tip: In exams, carefully underline each individual condition in a worded statement before writing the logic expression.

4.02 Boolean Operators

The three basic Boolean operators are:

  • AND: TRUE only if both inputs are TRUE.
  • OR: TRUE if at least one input is TRUE.
  • NOT: TRUE if the input is FALSE (inverts value).

Additional operators you must know:

  • NAND: TRUE unless both inputs are TRUE.
  • NOR: TRUE only if both inputs are FALSE.
  • XOR: TRUE if exactly one input is TRUE (but not both).
