2.01 Types of Data Transmission

Data transmission refers to how data moves from one device to another. It can be categorized in two main ways:

  • Serial Transmission: Bits are sent one after another over a single channel.
  • Parallel Transmission: Multiple bits are sent simultaneously across multiple channels.

2.02 Direction of Data Transmission

Communication channels may allow one-way or two-way data flow:

  • Simplex: Data flows in one direction only (e.g., keyboard to computer).
  • Half-Duplex: Data flows both ways, but only one direction at a time (e.g., walkie-talkies).
  • Full-Duplex: Data flows in both directions simultaneously (e.g., telephone).

2.03 Synchronisation

For data transmission to be successful, the sender and receiver must be synchronized:

  • Synchronous: Data is sent with a clock signal to keep sender and receiver aligned. Efficient for continuous streams.
  • Asynchronous: Data is sent one byte at a time with start and stop bits. Useful for irregular data transfer.

2.04 Transmission Errors

Errors occur when bits are corrupted during transmission due to noise or interference.

Common methods to detect and correct errors include:

  • Parity Checking: Adding a parity bit (odd or even) to detect errors.
  • Checksums: Blocks of data summed and compared at the receiver.
  • Cyclic Redundancy Check (CRC): Uses polynomial division to detect errors with high accuracy.
Worked Example: If the parity scheme is odd, and the data is 1010001, the parity bit should be 0 (making total 5 ones, odd).

2.05 Automatic Repeat Requests (ARQ)

ARQ is a protocol for ensuring reliable transmission:

  • Acknowledgement (ACK): Receiver confirms receipt.
  • Timeout: If no ACK is received in time, sender retransmits.
  • Negative Acknowledgement (NAK): Receiver signals error, prompting retransmission.

2.06 Transmission Mediums

Data can be transmitted through various mediums:

  • Copper Cable (Twisted Pair, Coaxial): Cheap, but susceptible to interference.
  • Optical Fibre: High bandwidth, secure, immune to interference.
  • Wireless: Uses radio/microwave, convenient but less secure.

2.07 Packet Switching

Data is split into packets before transmission. Each packet contains:

  • Header: Source, destination, sequence number, and error detection info.
  • Payload: Actual data being transmitted.

2.08 Encryption

Encryption secures data by transforming it into ciphertext:

  • Symmetric Encryption: Same key is used for both encryption and decryption. (e.g., AES)
  • Asymmetric Encryption: Public key encrypts, private key decrypts. (e.g., RSA)

2.09 Real-World Application

Data transmission concepts are applied in:

  • Secure online banking (encryption + ARQ)
  • Streaming services (packet switching, error detection)
  • Telecommunications (fibre optic and wireless systems)
